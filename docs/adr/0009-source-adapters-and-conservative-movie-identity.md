# Use Source Adapters and Conservative Movie Identity Matching

Easy Toho will model each external schedule website as a Schedule Source with its own adapter, currently TOHO Cinemas, SMT Cinemas, and T-Joy, and each adapter will return the same universal app-level schedule object at the Cinema + Selected Day grain. Source-specific fetch configuration, provider identifiers, URL conventions, raw response shapes, and parsing details stay inside the adapter layer; they are not part of the object consumed by the UI or shared aggregators. Universal source metadata may carry source identity for debugging, but the first universal model deliberately excludes booking, purchase, movie-detail, and source-detail URLs. Adapters own provider-specific fetching, parsing, language classification, format extraction, Showtime Availability mapping, source-label cleanup, source-specific date availability inside Easy Toho's fixed today-plus-six-days Planning Window, and a shared one-hour source cache policy. The UI should connect to one shared schedule surface and should not branch on a specific Schedule Source; aggregators only merge, sort, filter, and report failures across already-normalized Cinema schedules. New Cinemas from a working adapter, starting with Shinjuku Piccadilly and Shinjuku Wald 9, will participate in the existing single-cinema planner and cross-cinema movie and IMAX pages rather than being isolated behind a test-only view. The product name can remain Easy Toho, but page headings, empty states, and metadata that cover multiple Schedule Sources should use source-neutral copy instead of saying TOHO returned or owns every schedule. Movie detail routes will use the app-level Movie Identity rather than a source-local TOHO `mcode`, so a single `/movies/[movieId]` page can show matching screenings from multiple Schedule Sources together; old `/movies/{tohoMcode}` URLs do not need a compatibility layer during this prototype refactor. The shared model will use source-neutral availability values such as available, limited, sold out, not selling, and unknown for cross-source logic while also carrying a reader-facing availability label in the same universal field, so each Cinema can display clear provider wording without exposing provider-shaped status fields. Cross-source Movie Identity will be computed by the app, not trusted from provider IDs; the first matching rule will merge only when normalized English labels match and runtimes match, differ by only a few minutes, or one side is missing, with subtitled, dubbed, premium-format, and special-event variants folded into one Movie when the base film can be identified confidently. This favors avoiding false merges while testing new sources against the existing TOHO flow.
